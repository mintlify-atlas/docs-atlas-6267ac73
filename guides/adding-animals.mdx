---
title: "Adding animals"
description: "Add new animal endpoints to Floppa API"
---

All animals are defined in the `animalConfigs` slice in `backend/main.go`. Adding a new animal means adding one entry to that slice — the framework registers endpoints automatically.

## AnimalConfig struct

Each entry in `animalConfigs` uses this struct:

```go
type AnimalConfig struct {
    Endpoint    string // URL path (e.g., "capybara")
    Title       string // Display name
    Description string // Short description
    Animal      animals.Animal
}
```

The registration loop in `main.go` automatically creates:

- `GET /{endpoint}` — returns a random image
- `GET /{endpoint}/count` — returns the total image count
- An entry in `GET /api/animals` for the frontend to discover

<Tip>
  The frontend automatically picks up new animals from `/api/animals`. You don't need to modify any frontend code when you add a new animal.
</Tip>

<Note>
  Supported image formats for local animals: `.jpg`, `.jpeg`, `.png`, `.gif`, `.bmp`, `.webp`
</Note>

## Adding a PocketBase-backed animal

<Steps>
  <Step title="Create a PocketBase collection">
    In your PocketBase admin panel, create a new collection (e.g., `penguins`). Each record needs three fields:

    - `id` — auto-generated by PocketBase
    - `image` — file field for the image
    - `views` — number field for view tracking
  </Step>

  <Step title="Add the entry to animalConfigs">
    Open `backend/main.go` and add a new entry to the `animalConfigs` slice:

    ```go
    {
        Endpoint:    "penguin",
        Title:       "Penguin",
        Description: "Waddling pics",
        Animal: &animals.PocketBaseAnimal{
            Name:           "penguin",
            CollectionName: "penguins",
            PocketBaseURL:  cfg.PocketBaseURL,
        },
    },
    ```
  </Step>

  <Step title="Rebuild and restart the server">
    ```bash
    go build ./... && ./floppa-api
    ```

    Or with Docker:

    ```bash
    docker compose up --build
    ```
  </Step>

  <Step title="Test the new endpoint">
    ```bash
    curl http://localhost:8080/penguin
    ```

    You should receive a JPEG image in the response.
  </Step>
</Steps>

## Adding a local filesystem animal

<Steps>
  <Step title="Create a directory with image files">
    Add your images to a directory relative to the binary. For example:

    ```bash
    mkdir ./myfloppa
    cp my-images/*.jpg ./myfloppa/
    ```

    Supported formats: `.jpg`, `.jpeg`, `.png`, `.gif`, `.bmp`, `.webp`
  </Step>

  <Step title="Add the entry to animalConfigs">
    Open `backend/main.go` and add a new entry:

    ```go
    {
        Endpoint:    "myfloppa",
        Title:       "My Floppa",
        Description: "Local floppa pics",
        Animal: &animals.LocalAnimal{
            Name:      "myfloppa",
            Directory: "./myfloppa",
        },
    },
    ```
  </Step>

  <Step title="Add the directory to your Docker setup">
    If you're using Docker, copy the directory into the image or mount it as a volume.

    To copy into the image, add a line to your `Dockerfile`:

    ```dockerfile
    COPY ./myfloppa ./myfloppa
    ```

    To mount as a volume in `docker-compose.yml`:

    ```yaml
    volumes:
      - ./myfloppa:/app/myfloppa
    ```
  </Step>

  <Step title="Rebuild and test">
    ```bash
    go build ./... && ./floppa-api
    ```

    ```bash
    curl http://localhost:8080/myfloppa
    ```
  </Step>
</Steps>

## Verifying registration

After adding an animal, confirm it appears in the animals list:

```bash
curl http://localhost:8080/api/animals
```

The response includes your new animal alongside all existing ones:

```json
[
  { "endpoint": "floppapi", "title": "Floppa Generator 3000", "description": "Klikni a uvidis zazrak!" },
  { "endpoint": "penguin", "title": "Penguin", "description": "Waddling pics" }
]
```
