---
title: 'Self-hosting'
description: 'Run Floppa API directly on a server without Docker'
---

You can run Floppa API as a native binary on any Linux or macOS server. This approach gives you full control over the process and integrates cleanly with standard init systems like systemd.

## Prerequisites

- Go 1.24 or later
- [Bun](https://bun.sh/) (for building the frontend)
- A running [PocketBase](https://pocketbase.io/) instance

## Build and run

<Steps>
  <Step title="Build the frontend">
    From the repository root, install dependencies and build the frontend:

    ```bash
    cd frontend
    bun install
    bun run build
    ```

    This outputs static files to `frontend/dist/`. The backend serves this directory at startup.
  </Step>

  <Step title="Build the backend">
    Compile the Go binary from the `backend/` directory:

    ```bash
    cd backend
    go build -o floppa-api .
    ```

    This produces a `floppa-api` binary in the `backend/` directory.
  </Step>

  <Step title="Create config.json">
    Create `config.json` in the same directory as the binary. The binary searches for the config file at `/config.json`, `config.json`, and `./config.json` in that order.

    ```bash
    cp config.json.example config.json
    ```

    Edit `config.json` with your PocketBase connection details:

    ```json
    {
      "pocketbase_url": "http://localhost:8080",
      "pocketbase_admin": "admin@example.com",
      "pocketbase_password": "your_password"
    }
    ```
  </Step>

  <Step title="Run the binary">
    Start the server from the `backend/` directory:

    ```bash
    GIN_MODE=release ./floppa-api
    ```

    The API listens on port `8080`. The frontend is served from `./frontend/dist`.
  </Step>
</Steps>

## Production setup with systemd

For a persistent production deployment, create a systemd service unit. This example assumes the repository is deployed to `/opt/floppa-api`.

Create the file `/etc/systemd/system/floppa-api.service`:

```ini
[Unit]
Description=Floppa API
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/floppa-api
ExecStart=/opt/floppa-api/floppa-api
Environment=GIN_MODE=release
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Then enable and start the service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable floppa-api
sudo systemctl start floppa-api
```

Check the service status:

```bash
sudo systemctl status floppa-api
```

<Note>
  Set `WorkingDirectory` to the directory containing both the `floppa-api` binary and the `frontend/` folder. The binary serves `./frontend/dist` relative to its working directory.
</Note>

## Environment variables

| Variable   | Value     | Description                                    |
| ---------- | --------- | ---------------------------------------------- |
| `GIN_MODE` | `release` | Disables debug logging. Recommended for production. |
